# 🏪 YOLO26 Dense Retail Shelf Detection Pipeline

End-to-end object detection pipeline for densely packed retail shelf images using **YOLO26 Medium** with **SAHI** (Slicing Aided Hyper Inference).

## 🖥️ Target Hardware

| Component | Spec |
|-----------|------|
| **GPU** | NVIDIA RTX 4060 Ti (16 GB VRAM) |
| **RAM** | 32 GB System Memory |
| **CPU** | Intel Core i5-12400F (6 Cores) |

## 🔧 Key Technical Decisions

### Why SAHI Slicing for Training?

Your images are **~3000×4000 px** with up to **1,373 objects per image**. Training at `imgsz=640` would downscale each product to **~1–3 pixels** — making detection impossible. SAHI slicing preserves the native pixel resolution.

### Hardware Optimizations

- **AMP (FP16):** Halves VRAM usage → batch=16 fits comfortably in 16 GB
- **workers=6:** Matches your i5-12400F's 6 physical cores
- **cache=True:** RAM-caches dataset for faster epoch iterations (fits in 32 GB)
- **MuSGD optimizer:** YOLO26's native optimizer for faster convergence

### Dual SAHI Strategy

| Phase | Method | Purpose |
|-------|--------|---------|
| **Training** | Pre-slice into 640×640 tiles | Preserve small object resolution |
| **Inference** | SAHI sliding window on full-res images | Reconstruct full-image detections |
